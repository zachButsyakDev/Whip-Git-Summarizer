Git Summarizer:
Uses AI API calls to pass Git commit history alongside their respective files/messages to an AI to have it summarize to the user the changes in the codebase commit by commit.

MVP:
Exactly that. Parse all changes, commits, messages of 1 respective branch, pass all of that rawtext to an AI with the context of having it summarize changes.



HANDY STUFF TO USE:

Commit History	git rev-list <range>	Outputs raw hashes of commits; easy to loop through.
Commit Metadata	git show -s --format=...	Provides full control over output (e.g., JSON-like or pipe-separated).
File Changes	git diff --name-only <old> <new>	Tells you which files changed without the noise of the diff itself.
Actual Code Diffs	git diff <old> <new>	Provides the raw patch data for the AI to analyze.

Get "what's new"	git rev-list --reverse <last_seen>..HEAD
Get commit info	git show -s --format="%s"
Get the actual change	git show <hash>
Check file stats	git diff --stat <hash>^ <hash>
Update state	git rev-parse HEAD > .git/last_viewed
